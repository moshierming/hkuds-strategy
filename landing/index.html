The Problem

Traditional RAG breaks when documents are interconnected

Vector similarity finds what sounds similar. It can't find why Document A and Document C are related — or trace reasoning across 50 connected clauses in a legal contract.

❌ Traditional RAG

  • Treats every document as isolated chunks
  • Finds similar text — misses relationships
  • Loses context when answers span multiple documents
  • No global understanding of your corpus
  • Answers degrade as knowledge base grows

✅ LightRAG

  • Builds entity-relationship graph from your documents
  • Dual-mode: Local (precise) + Global (holistic) queries
  • Traces multi-hop reasoning across your entire corpus
  • Understands why documents are related
  • Gets smarter as your knowledge base grows
How it works

Up and running in 3 steps

1

Index your documents

Drop in PDFs, Word docs, Markdown, or plain text. LightRAG extracts entities and relationships and builds a knowledge graph — automatically.

2

Choose your query mode

Local — precise answers from specific entities.
Global — holistic understanding across the entire corpus.
Hybrid — combines both for maximum accuracy.

3

Get relationship-aware answers

LightRAG retrieves from both the KG and vector store simultaneously, fusing graph reasoning with semantic similarity for answers your users can trust.

# Install
pip install "lightrag-hku[api]"

# Python API — 5 lines to production-grade RAG
from lightrag import LightRAG, QueryParam

rag = LightRAG(working_dir="./knowledge_base")

await rag.ainsert(your_documents)  # index once

answer = await rag.aquery(
    "What are all the obligations triggered by clause 12.3?",
    param=QueryParam(mode="hybrid")
)
