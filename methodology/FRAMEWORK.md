# 研究方法论框架 · Research Methodology Framework

> 不是一次性分析，而是一套持续运转的多源情报引擎。  
> HKUDS 是坐标原点，不是边界。

---

## 核心理念

**永不封闭的信号网络**

HKUDS 是我们深耕的一块土地——但眼光绝不能只盯着它。  
ProductHunt 上的产品、HackerNews 上的讨论、GitHub Trending 上的爆款、  
arXiv 上的论文、Twitter 上的争论——任何地方都可能出现改变判断的信号。

**四层持续循环**

```
收集信号 → 三角验证 → 形成洞察 → 执行行动 → 回顾反思 → 收集信号...
```

1. **信号层**（自动化）：多渠道并行采集，不依赖单一来源
2. **验证层**（人工判断）：交叉比对，过滤噪音，找到真正的信号
3. **洞察层**（原创思考）：基于数据形成自己的判断，写下来
4. **行动层**（物化输出）：每个洞察至少导向一个具体可执行物

**关键原则**：信号来自外部，洞察来自自己。不要转发信息，要形成判断。

## 五维度分析法

对任何 AI/ML 项目，从以下 5 个视角同时切入：

### 维度 1：技术本质（Technology）
> 这个东西真正在解决什么计算问题？

| 分析项 | 关键问题 |
|--------|---------|
| 核心算法 | 创新点在哪？和 SOTA 的差距是什么？ |
| 工程可落地性 | 能在 A100 以外跑吗？推理成本如何？ |
| 依赖链 | 依赖哪些上游（PyTorch/HF/etc）？有无脆弱点？ |
| 可扩展性 | 到 10x 规模时架构会崩溃吗？ |
| 代码质量 | PR 合并速度、测试覆盖、文档完整度 |

### 维度 2：市场位置（Market）
> 这个问题的市场有多大？谁在付钱？

| 分析项 | 关键问题 |
|--------|---------|
| TAM/SAM | 可服务市场规模（要用数字，不要形容词） |
| 付费意愿 | 谁会为这个付钱？为什么现在付？ |
| 竞品格局 | 直接竞品 / 替代方案 / 内部自建可能性 |
| 分发渠道 | 用户从哪里发现并采用这类工具？ |
| 监管风险 | 数据隐私 / 行业合规 / 地缘政治壁垒 |

### 维度 3：人与组织（People）
> 谁在做？他们能做成吗？

| 分析项 | 关键问题 |
|--------|---------|
| 核心作者 | 发表记录 / 引用量 / 工业经验 |
| 活跃贡献者 | 核心 contributor 数量与质量 |
| 机构背景 | 导师 / 实验室 / 资助方 |
| 商业化意愿 | 有创业迹象吗？受 licensing 吗？ |
| 社区健康度 | issue 响应速度 / Discord 活跃度 |

### 维度 4：时机与势（Timing）
> 为什么是现在？

| 分析项 | 关键问题 |
|--------|---------|
| 技术使能因素 | 哪个上游突破使这成为可能？（GPT-4 / CLIP / etc）|
| Star 增长曲线 | 是线性还是指数？拐点在哪？ |
| 媒体/学界关注 | arXiv 引用趋势 / Twitter 热度 |
| 竞品动向 | 大公司有没有开始做类似的？ |
| 行业痛点时间窗 | 这个痛点还会存在多久？ |

### 维度 5：个人行动机会（Own Action）
> 我能做什么？

| 行动类型 | 具体方式 |
|---------|---------|
| **贡献** | Bug fix / 文档 / 新 feature / 测试 |
| **延伸** | 上层应用 / 集成工具 / 垂直方向 |
| **发布** | 博客 / 视频 / ProductHunt / 技术社区 |
| **接触** | PR / Issue / 邮件作者 / 加入路线图讨论 |
| **商业化** | SaaS wrapper / 咨询 / 托管版本 |

## 分析优先级矩阵

```
                高商业价值
                    │
    B 机会          │    A 优先出击
    （养着观察）     │   （立即行动）
                    │
低技术壁垒 ─────────┼──────────── 高技术壁垒
                    │
    D 跳过          │    C 学习素材
                    │   （技术储备）
                低商业价值
```

**分级规则**：
- **A 级**：技术壁垒高（难复制）× 商业价值高（有人付钱）→ **立即行动**
- **B 级**：低壁垒 × 高价值 → 市场过热，寻找差异化切入点
- **C 级**：高壁垒 × 低价值 → 作为技术储备，等待应用场景
- **D 级**：低壁垒 × 低价值 → 不投入

## 研究节奏

| 周期 | 行动 | 工具 |
|------|------|------|
| **每天** | 浏览 arXiv cs.AI + cs.IR 新论文 | `automation/arxiv_monitor.py` |
| **每周一** | 拉取 HKUDS 上周 GitHub 动态 | `automation/fetch_github_stats.py` |
| **每周五** | 生成本周研究摘要 | `automation/generate_digest.py` |
| **每月** | 更新 PROJECT_MAP.md 评级 | 手动 + 辅助脚本 |
| **随时** | 记录灵感 → 填入 Opportunity Log | `templates/OPPORTUNITY_LOG.md` |

---

## 行动输出标准

每一个研究循环应至少产出以下之一：

1. **一个新洞察** → 写入 `analysis/INSIGHTS.md`
2. **一个机会记录** → 填入 `logs/opportunities.md`
3. **一个具体行动** → PR / blog post / 工具代码 / 联系人
4. **一个认知更新** → 修正之前的判断并说明原因

> 没有输出 = 没有研究。思考必须物化。

## 关键原则

1. **多角度 > 深度**：先用 5 维度框架扫描，再对 A 级目标深潜
2. **具体 > 抽象**：所有结论必须有数字或代码支撑
3. **行动 > 分析**：每个研究循环必须产出可执行物
4. **自动化 > 手动**：重复性数据收集全部脚本化
5. **迭代 > 完美**：30 分钟的粗略分析 > 永远写不完的完美分析
